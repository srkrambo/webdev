Week 1: Structural Elements

Beyond giving meaning to text, HTML provides generic elements that are crucial for structuring the layout of your page. These elements act as containers that you can style with CSS.

Core Concepts

  • Block vs. Inline:
    • Block-level elements (like <div>, <p>, <h1>) always start on a new line and take up the full width available.
    • Inline elements (like <span>, <a>, <strong>) do not start on a new line and only take up as much width as necessary.
  • Generic Containers:
    • <div>: The primary block-level container. It's used to group large sections of content, like a header, sidebar, or footer, which you can then style.
    • <span>: The primary inline container. It's used to group a small piece of text or an element within a larger block, often to apply a specific style.
  • id and class: These attributes are your primary hooks for styling with CSS and selecting with JavaScript.
    • id: A unique identifier for one element on a page (e.g., id="main-logo").
    • class: A reusable identifier that can be applied to multiple elements (e.g., class="news-article").
  • Comments: You can leave notes in your code that the browser will ignore using <!-- This is a comment -->. This is great for staying organized.
