Week 1: Lists & Tables

Lists and Tables are used to organize and display related data in a structured way. Lists are great for simple collections of items, while tables are perfect for complex, grid-based data.

Core Concepts: Lists

  • Unordered List <ul>: Used for a list of items where the order doesn't matter. It uses <li> (list item) tags for each item and typically displays them with bullet points.
  • Ordered List <ol>: Used for a list where the order is important (like a recipe or a countdown). It also uses <li> tags and displays them with numbers.
  • Description List <dl>: A list of terms (using <dt>) and their descriptions (using <dd>). It's great for glossaries or metadata.

Core Concepts: Tables

  • Table Structure: Tables are defined with the <table> tag. Inside, you can have a <thead> for the header row(s), a <tbody> for the main data, and a <tfoot> for footer rows.
  • Rows & Cells: Each row is created with a <tr> (table row). Inside a row, header cells are created with <th> (table header) and standard data cells are created with <td> (table data).
  • Spanning: You can merge cells across columns with the colspan attribute and across rows with the rowspan attribute (e.g., <td colspan="2">).
